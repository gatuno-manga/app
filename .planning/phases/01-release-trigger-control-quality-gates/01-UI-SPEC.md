## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, dense inline spacing |
| sm | 8px | Input/helper/error text spacing |
| md | 16px | Default control padding and vertical rhythm |
| lg | 24px | Group spacing between validation/concurrency sections |
| xl | 32px | Card/container internal padding |
| 2xl | 48px | Major section separation |

Exceptions: Minimum touch target height remains 44px for icon-only or compact controls.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Validate Release Tag |
| Empty state heading | No release runs yet |
| Empty state body | Create and push a valid tag in `vMAJOR.MINOR.PATCH` format to start the release flow. |
| Error state | Release validation failed. Fix the tag format or quality gate errors (`flutter analyze`/`flutter test`) and rerun. |
| Destructive confirmation | Cancel active release run: “Cancel this running release for this version? The newest run will continue per concurrency policy.” |

Source: `01-CONTEXT.md` decisions D-01..D-07 + phase defaults.

---

## Visual Hierarchy

- **Primary focal point:** Card/section with CTA **"Validate Release Tag"**.
- **Secondary focal point:** Validation status timeline/result states (pass/fail and reason).
- **Tertiary elements:** Concurrency policy details and safety notes.

This ordering ensures users first understand whether release can proceed, then why it failed/succeeded, then what governance rules apply.
